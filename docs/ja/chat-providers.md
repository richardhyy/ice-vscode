# チャットプロバイダー

ICEは複数のLLMプロバイダーを標準でサポートしており、ニーズに最適なAIモデルを柔軟に選択できます。

## 組み込みプロバイダー

- VS Code Language Models（GitHub CopilotなどVS Codeですでに利用できるモデル。**APIキーの入力は不要**）
- OpenAI（GPTモデル、例：GPT-4o、GPT-4o mini）
- Anthropic（Claudeモデル、例：Claude Sonnet 4.5、Claude Opus 4.1）
- Google（Geminiモデル、例：Gemini 2.5 Flash、Gemini 2.5 Pro）
- Poe（単一のキーで数百のモデルとボット）

## VS Code Language Models（APIキー不要）

**VS Code Language Models** プロバイダーは、GitHub Copilotのサブスクリプションや他の拡張機能が提供するモデルなど、VS Code内ですでに利用できるモデルと通信します。認証はVS Codeが管理するため、APIキーもエンドポイントも入力する必要はありません。

- プロバイダーピッカーから **VS Code Language Models** を選択します。ICEが最初にメッセージを送信する際、VS Codeがモデルの使用許可を求めるので、**許可**を選択してください。
- プロバイダー設定を開くと、VS Codeが提供するモデルから特定の **Model** を選べます。空欄のままにするとICEが自動的に選択します。一覧は、インストール済みでサインインしているものに合わせて更新されます。
- 一覧が空の場合は、GitHub Copilotにサインインする（または言語モデルを提供する拡張機能をインストールする）と表示されます。
- ツール呼び出しに対応しているため、他のプロバイダーと同様にICEのツールやMCPサーバーと連携できます。

> このプロバイダーにはVS Code 1.95以降が必要です。

## プロバイダーの使用方法

1. 新しい`.chat`ファイルを作成するか、インスタントチャットセッションを開始します。
2. ドロップダウンメニューから希望のプロバイダーを選択します。
3. プロンプトが表示されたらAPIキーを入力します（初回使用時のみ）。VS Code Language Models プロバイダーはキー不要で、VS Codeがアクセスを管理します。

## カスタムプロバイダー

ICEでは、追加のLLMやAPIサービス用にJavaScriptベースのカスタムプロバイダーを作成できます。カスタムプロバイダーを追加するには：

1. プロバイダーピッカーを開きます。
2. 「カスタムプロバイダーフォルダを開く」をクリックします。
3. カスタムプロバイダースクリプト（.js）を「providers」ディレクトリに貼り付けます。

カスタムプロバイダーの作成に関する詳細情報は、[カスタムプロバイダー](ja/custom-providers.md)を参照してください。

!> **セキュリティ警告**：チャットプロバイダーは任意のコードを実行する可能性があり、APIキーを第三者に公開する可能性があります。信頼できるソースからのプロバイダーのみを使用し、実行前に必ずコードを確認してください。

## プロバイダーの切り替え

インライン設定を使用して、会話の途中でプロバイダーを切り替えることができます。これにより、1つのチャットセッション内で異なるモデルの強みを活かすことができます。
