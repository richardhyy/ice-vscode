# ビルトインプロバイダの設定

ICEには複数のビルトインプロバイダが付属しており、それぞれに独自の設定オプションがあります。これらの設定を理解することで、異なるLLMプロバイダとのやり取りを微調整できます。このガイドでは、OpenAI互換プロバイダを例に使用して、共通の設定変数について説明します。

## OpenAI互換プロバイダの設定

OpenAI互換プロバイダは、OpenAIのAPIや類似のサービスと連携するように設計されています。以下が主要な設定変数です：

### 必須変数

- `APIKey`: APIサービスの認証キー。**Ollama** や **LM Studio** などのローカルプリセットではキーは不要です。入力を求められた場合は任意のプレースホルダー（例：`ollama`）を入力してください。

- `Preset`: 接続するAPIサービス。ホストとパスを手動で組み立てる代わりに、よく知られたOpenAI互換サービスを選ぶだけで、ICEが正しいエンドポイントを補完します。下記の[プリセット](#プリセット)を参照してください。

- `Model`: 使用する特定の言語モデル（例：`gpt-4o-mini`）。異なるモデルには異なる能力とパフォーマンス特性があります。このフィールドはエンドポイントが提供するモデルを一覧表示できます。リストから選ぶか、任意のモデルIDを入力してください。

- `MaxTokensToSample`: モデルが生成すべき最大トークン数。これは出力の長さを制御するのに役立ちます。

- `SystemPrompt`: AIアシスタントの振る舞いや役割を設定するプロンプト。AIの応答のコンテキストとパーソナリティを定義するのに役立ちます。

### プリセット

`Preset` 変数は、よく知られたOpenAI互換サービスを選択します。これにより、ホスト・パス・ヘッダーを手動で設定する必要がなくなります。使用するサービスを選ぶだけです：

| プリセット | エンドポイント | APIキー |
| --- | --- | --- |
| `OpenAI` | `https://api.openai.com/v1` | 必要 |
| `Ollama` | `http://localhost:11434/v1` | 不要（ローカル） |
| `LM Studio` | `http://localhost:1234/v1` | 不要（ローカル） |
| `OpenRouter` | `https://openrouter.ai/api/v1` | 必要 |
| `Groq` | `https://api.groq.com/openai/v1` | 必要 |
| `DeepSeek` | `https://api.deepseek.com/v1` | 必要 |
| `Together` | `https://api.together.xyz/v1` | 必要 |
| `Mistral` | `https://api.mistral.ai/v1` | 必要 |
| `xAI` | `https://api.x.ai/v1` | 必要 |
| `Fireworks` | `https://api.fireworks.ai/inference/v1` | 必要 |
| `Perplexity` | `https://api.perplexity.ai` | 必要 |
| `Custom` | `BaseURL` を自分で設定 | サービスによる |

一覧にないエンドポイント（自己ホスト型サーバー、プロキシ、新しいサービスなど）の場合は、`Custom` を選び、オプション変数 `BaseURL` を設定してください（下記参照）。

### オプション変数

- `BaseURL`: エンドポイントのOpenAI形式のベースURL（例：`https://api.openai.com/v1`）。バージョンセグメントは含みますが、`/chat/completions` は含みません。空のままにすると選択した `Preset` のURLが使われます。カスタム・自己ホスト型・プロキシ経由のサービスを指定する場合に設定します。設定するとプリセットより優先されます。

- `Temperature`: モデルの出力のランダム性を制御する0から1の間の値。高い値（例：0.8）は出力をよりランダムにし、低い値（例：0.2）はより焦点を絞った決定論的なものにします。

- `LogitBias`: 出力に特定のトークンが現れる可能性を調整できるJSONオブジェクト。特定の単語やフレーズを促進または抑制するために使用できます。

- `AdditionalHeaders`: APIリクエストに含める追加のHTTPヘッダーを指定するJSONオブジェクト。カスタム認証スキーム（例：OpenRouterのランキングヘッダー）やその他のAPI固有の要件に役立ちます。

## 設定変数の理解

### API設定
- `APIKey`、`Preset`、（任意で）`BaseURL` が協力してLLMサービスへの接続を確立します。`Preset` が既知のエンドポイントを選択し、`BaseURL` がカスタムサービス用にそれを上書きし、`APIKey` がリクエストを認証します。ローカルプリセットではキーは不要です。

### モデルの動作
- `Model`は使用する特定のAIモデルを選択します。異なるモデルは、能力、知識のカットオフ日、パフォーマンス特性が異なる場合があります。
- `SystemPrompt`はAIの初期コンテキストを設定し、会話中に想定する「パーソナリティ」や役割を効果的に与えます。

### 出力制御
- `MaxTokensToSample`はAIの応答の長さを制限します。これはコストを管理し、簡潔な回答を確保するのに役立ちます。
- `Temperature`は出力の創造性とランダム性に影響します。低い値は事実に基づいた予測可能な応答に適していますが、高い値はより創造的で多様な出力につながる可能性があります。
- `LogitBias`はモデルのトークン選択プロセスを細かく制御でき、出力のスタイルや内容を導くために使用できます。

### 高度な使用法
- `BaseURL` を使うと、ビルトインのプリセットにないOpenAI互換エンドポイント（自己ホスト型サーバー、社内プロキシ、新しいサービスなど）にプロバイダを向けることができます。
- `AdditionalHeaders`は、異なるAPI実装との連携やリクエストにカスタムメタデータを追加するための柔軟性を提供します。

## ベストプラクティス

1. **APIキーのセキュリティ**: `APIKey`は常に安全に保管し、決して公開しないでください。

2. **カスタマイズ**: 異なる`SystemPrompt`値を試して、AIの動作を特定のユースケースに合わせてカスタマイズしてください。

3. **パフォーマンスチューニング**: `Temperature`と`MaxTokensToSample`を調整して、応答の品質、長さ、生成速度のバランスを取ってください。

4. **コスト管理**: `MaxTokensToSample`の設定に注意してください。高い値はAPI使用量とコストを増加させる可能性があります。

5. **互換性**: 使用するサービスに合った `Preset` を選ぶか、それ以外の場合は `Custom` を選んで `BaseURL` を設定してください。いずれの場合も、指定する `Model` がそのサービスで提供されていることを確認してください。

> **以前のバージョンからのアップグレード** 接続は `APIHost` と `APIPath` の代わりに `Preset` と `BaseURL` で設定するようになりました。デフォルトのOpenAIエンドポイントを使用していたプロバイダはそのまま動作します。以前 `APIHost` をカスタムまたは自己ホスト型サービスに向けていた場合は、一度だけ対応するプリセット（または `Custom` + `BaseURL`）を選び直してください。`APIHost`/`APIPath` を設定したインライン `#config` ブロックは引き続き動作します。
